# 10 - Kali Linux & Security Onion Network Detection Lab

## Overview
This folder documents how I integrated Kali Linux with Security Onion to practice real network detection, packet analysis, and alert investigation. It simulates how SOC analysts detect, analyze, and respond to suspicious network activity in enterprise environments.

## What's Included

### Security Onion Setup & Configuration
- Security Onion installation & setup
- Network sensor configuration
- Connecting Kali Linux as an attacker/test machine

### Attack Simulation & Traffic Generation
- Generating controlled network activity:
  - Network scans (nmap, etc.)
  - Ping sweeps
  - SSH attempts
  - Web traffic generation

### Alert Analysis & Investigation
- Reviewing alerts in Security Onion
- Investigating suspicious activity
- Documenting findings

### Tools & Technologies Used
- **Zeek logs** - Network metadata and connection logging
- **Suricata alerts** - Intrusion detection system alerts
- **PCAP analysis** - Packet capture examination
- **Kibana dashboards** - Visualization and log analysis

## What I Did

I installed Security Onion as a network monitoring sensor and connected Kali Linux to generate controlled attack traffic. I tested scans, SSH attempts, and web requests, then analyzed the alerts generated by Suricata and Zeek. I reviewed PCAPs, dashboards, and logs to understand how network activity becomes alerts.
