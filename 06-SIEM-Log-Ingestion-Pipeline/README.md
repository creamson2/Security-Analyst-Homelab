## 06 – SIEM Log Ingestion Pipeline
This folder documents how I built and tested a log ingestion pipeline for my Security Analyst Lab.
It covers how logs move from endpoints → to the SIEM → to dashboards for analysis.

## What’s included
- Log sources (Kali, Windows, Linux, tools)

- How logs are collected and forwarded

- Wazuh → Elasticsearch → Dashboard flow

- Splunk ingestion testing

- Parsing, normalization, and indexing notes

- Troubleshooting ingestion failures

## Purpose
This folder shows how I design and maintain a working SIEM pipeline.
It demonstrates my understanding of log flow, parsing, indexing, and how logs become alerts — core skills for any Security Analyst.
